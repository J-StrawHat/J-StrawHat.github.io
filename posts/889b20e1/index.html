¶B. 明码 #STL

¶题意

把每个字节转为2进制表示，1表示墨迹，0表示底色。每行2个字节，一共16行，布局是：

给定一段由多个汉字组成的信息，每个汉字用323232个字节表示，这里给出了字节作为有符号整数的值。题目的要求隐藏在这些信息中。你的任务是复原这些汉字的字形，从中看出题目要求，并根据要求填写答案。

¶分析

注意题目中说明是有符号二进制，也就说如果某个十进制数为负数，则它的二进制为补码（对于负数的补码，最高位为符号位，要取1，对原码取反再加一）。

题目简而言之，即为每一行有323232个数字，每两个数字组成一行的布局。比如

关于十进制对二进制的转换，可以使用STL的bitset数据结构，十分方便，初始时定义为8位的二进制串，每次输入一个十进制，它就能将该数值转换为二进制串，同时还能将这个二进制串输出来！

¶C. 乘积尾零 #因数分解

¶题意

如下的10行数据，每行有10个整数，请你求出它们的乘积的末尾有多少个零？

¶分析

注意审题，题目要求的是乘积结果的尾部，也就说它并不是问你乘积结果整个数字含有多少个零！（如果是求这个的话，难度++）

观察到，乘积结果的尾部有xxx个000，说明该乘积结果含有xxx个因子101010，也就说至少含有xxx个因子555和xxx个因子222（多余的因子成单，没有与之对应的因子进行配对），由此我们可以将所有数中的因子555和222分解出来，统计含222和555的因子数量并取两者最少值，即能推出乘积结果尾部000的数量。

举个栗子，对于1050=2×3×7×5×51050=2\times3\times7\times5\times51050=2×3×7×5×5（含两个555，一个222），而180=9×2×2×5180=9\times2\times2\times5180=9×2×2×5（含一个555，两个222），则他们的乘积为1050×180=189000=(9×3×7)×(5×2)×(5×2)×(5×2)1050\times180=189000=(9\times3\times7)\times(5\times2)\times(5\times2)\times(5\times2)1050×180=189000=(9×3×7)×(5×2)×(5×2)×(5×2)（即三对5×25\times25×2构成尾部的三个000）

¶F. 递增三元组 #二分思想 #前缀和

¶题意

给定三个整数数组A = [A1, A2, ... AN], B = [B1, B2, ... BN],
C = [C1, C2, ... CN]（N≤1e5N\leq1e5N≤1e5），请你统计有多少个三元组$(i, j, k) $同时满足：

  • 1≤i,j,k≤N1 \leq i, j, k \leq N1≤i,j,k≤N

  • Ai<Bj<CkA_i < B_j < C_kAi​<Bj​<Ck​

¶分析

不妨对这三个整数数组先升序排序，我们知道，要确定一个三元组(i,j,k)(i,j,k)(i,j,k)，那么可以先从AAA中取一个元素aia_iai​，再从BBB中找出第一个大于等于aia_iai​的元素bjb_jbj​，同理再从CCC中找出第一个大于等于bjb_jbj​的元素ckc_kck​。但如果这样直接枚举每个i,j,ki,j,ki,j,k一定会超时。

我们枚举每个bjb_jbj​，通过二分确定了第一个ckc_kck​的位置pospospos，可以知道有(n−pos)(n-pos)(n−pos)个C数组元素，大于等于bjb_jbj​。我们在枚举bjb_jbj​并确定pospospos的过程中，同时用前缀和维护前jjj个元素bjb_jbj​，有多少个小于C数组的元素(sum[j]=∑1jn−possum[j]=\sum_{1}^j{n-pos}sum[j]=∑1j​n−pos)。

接着，枚举每个$ a_i，通过二分确定了第一个，通过二分确定了第一个，通过二分确定了第一个b_j的位置的位置的位置posB，查询，查询，查询sum[n]-sum[posB]$，可以知道满足题目要求的三元组个数。

¶G. 螺旋折线 #性质观察 #等差数列

¶题意

如下图所示的螺旋折线经过平面上所有整点恰好一次，对于整数点(x,y)(x,y)(x,y)（−1e9≤x,y≤1e9-1e9\leq x, y\leq 1e9−1e9≤x,y≤1e9），定义它到原点的距离：dix(x,y)dix(x,y)dix(x,y)，为从原点到(x,y)(x, y)(x,y)的螺旋折线段的长度。

¶分析

一开始我按路径递增去模拟，只要转向次数为偶数次，就对前进长度+1，但还是超时，下面就是我的75分qaq。

参考了@虎老狮的思路。我们可以将折线每一行的左下角长度为111的线段顺时针旋转90度（见下图），于是螺旋折线就变成长度易求的正方形环条了。于是点在螺旋折线上的移动就转化为一个点从每一个正方形环的左下角作为起点，顺时针移动，移动整一个环后，便直接跳到下一个更长的正方形环的左下角起点，…. 。容易观察到每一条正方形环的周长组成了以0为首项（即第一个正方形环是空的），以8为公差的等差数列。

如上图，我们假设点A(xi,yi)(x_i,y_i)(xi​,yi​)在该正方形对角线上侧，那么题目要求的dis(xi,yi)dis(x_i, y_i)dis(xi​,yi​)，实际上就是（第三条边左下角起点出发到达点(xi,yi)(x_i,y_i)(xi​,yi​)的距离）+（不包括第三条边的内部正方形的周长之和）。由等差数列求和公式，内部边的长度之和就是Si=(0+(i−1)×8)×(i−1)2S_i = \frac{(0 + (i-1)\times8)\times(i-1)}{2}Si​=2(0+(i−1)×8)×(i−1)​。第三条边左下角起点出发到达点(xi,yi)(x_i,y_i)(xi​,yi​)的距离，就是(i+xi)+(i+yi)(i+x_i)+(i+y_i)(i+xi​)+(i+yi​)。如果是位于正方形对角线下侧的点B(xj,yj)(x_j,y_j)(xj​,yj​)，它需要用第三条边的周长减去(j+xj)+(j+yj)(j+x_j)+(j+y_j)(j+xj​)+(j+yj​)
